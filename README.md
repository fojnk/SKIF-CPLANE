# Control Plane

Control Plane - веб-платформа для планирования, запуска и анализа экспериментов на цифровых двойниках станций ЦКП СКИФ.
Система объединяет пользователей, модели, параметры экспериментов, историю изменений и логи в одном интерфейсе.

## Зачем нужна платформа

Control Plane решает прикладную задачу управления жизненным циклом вычислительного эксперимента:

- уменьшает время от постановки задачи до первого запуска (Time-to-Experiment);
- обеспечивает воспроизводимость результатов за счет версий и истории изменений;
- централизует управление доступом (рабочие зоны, роли, права);
- дает наблюдаемость выполнения экспериментов в реальном времени;
- упрощает совместную работу исследователей, инженеров и студентов.

## Для кого

- исследователи и инженеры, работающие с цифровыми двойниками;
- студенты и младшие научные сотрудники для учебных и прикладных сценариев;
- администраторы платформы, управляющие пользователями, ролями и окружением.

## Что внутри

- `frontend` - клиентское приложение (React + TypeScript);
- `backend` - API и бизнес-логика (Golang, Gin, WebSocket);
- `postgres` - хранение пользователей, параметров, версий, логов;
- `nginx` - единая точка входа в веб-интерфейс и API.

## Как использовать платформу

Типовой сценарий работы:

1. Войти в систему под своей учетной записью.
2. Выбрать рабочую зону/проект и настроить модель эксперимента.
3. Задать параметры и запустить эксперимент.
4. Следить за состоянием выполнения в реальном времени.
5. Анализировать результаты, сравнивать версии, при необходимости откатываться.
6. Использовать журнал действий и логи для аудита и диагностики.

## Деплой (Docker Compose)

### Требования

- Docker Engine и Docker Compose Plugin;
- репозиторий **skif_platform_supervisor** рядом с CPLANE: `../skif_platform_supervisor` (нужен сборке `java-supervisor` в `docker-compose.yml`);
- свободные порты **`8080`** (вход в UI), **`8081`** (HTTP супервизора), при желании **`5672`** / **`15672`** (RabbitMQ).

### Быстрый старт

Из корня этого репозитория:

```bash
make up
```

Команда создаёт каталог `SUPERVISOR_HOST_WORKDIR` (по умолчанию `/tmp/supervisor-work`), пересобирает образы локальных сервисов и поднимает **весь** стек из `docker-compose.yml`: PostgreSQL, RabbitMQ, backend, **java-supervisor**, nginx. Миграции БД подтягиваются при старте backend.

После запуска веб-интерфейс CPLANE: **`http://localhost:8080`**.

### Полезные команды

```bash
make help      # список доступных целей
make ps        # статус контейнеров
make logs      # логи сервисов
make down      # остановка и удаление контейнеров
make restart   # обновить и пересоздать окружение
make rebuild   # полная пересборка без кэша
make clean     # удалить контейнеры, volume и локальные образы
```

### Структура сервисов в окружении

- `postgres` — БД;
- `rabbitmq` — очередь AMQP для backend и супервизора (`http://localhost:15672`, пользователь и пароль: `cplane` / `cplane`);
- `backend` — API (миграции применяются при старте);
- `java-supervisor` — исполнитель пайплайнов (образ собирается из `../skif_platform_supervisor/development/supervisor`, статусы: `http://localhost:8081`);
- `nginx` — внешний HTTP-вход (`8080 -> 80`).

Переменная **`SUPERVISOR_HOST_WORKDIR`** (можно переопределить перед `make up`) задаёт каталог на хосте для JSON-файлов супервизора; см. комментарии в `docker-compose.yml`.

## Локальная разработка frontend

Для пересборки фронтенд-бандла:

```bash
make frontend-rebuild
```
