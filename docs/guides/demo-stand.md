# Демо-данные тестового стенда

Указатель всей документации: [`docs/README.md`](../README.md).

После применения миграций БД **`000008_demo_stand_seed`** создаётся готовая связка для проверки UI и полного контура с RabbitMQ и Java-супервизором.

## Что появляется в базе

| Сущность | Имя / маркер |
|----------|----------------|
| Неймспейс | `demo-stand-skif` |
| Проект | **Демо-проект оптической цепочки** |
| Шаблон эксперимента | **Демо пайплайн (супервизор)** |
| Эксперимент | один активный пайплайн в этом проекте (`status`: `idle`) |
| Датасеты | **Демо — параметры пучка**, **Демо — выход детектора (шаблон)** |
| Связи эксперимент ↔ датасеты | алиасы `beam_input`, `detector_output` |
| Переменная эксперимента | `demo_notes` (строка с пояснением) |

Конфигурация шаблона — JSON с массивом **`models`** в формате Java-супервизора (два шага: `PYTHON`, затем `C`), как в [`backend/json/supervisor_experiment.example.json`](../../backend/json/supervisor_experiment.example.json).

Миграция **идемпотентна**: если неймспейс `demo-stand-skif` уже есть, повторный прогон только пишет NOTICE в лог PostgreSQL и не дублирует строки.

## Доступ и запуск полной связки

1. Войти под локальной учётной записью с правами администратора разработки (пользователи `root` / `admin` из [`backend/config.local.yaml`](../../backend/config.local.yaml) и миграции RBAC имеют роль `root_admin`).
2. Открыть неймспейс **demo-stand-skif** → проект **Демо-проект оптической цепочки** → эксперимент **Демо пайплайн**.
3. Для проверки супервизора поднимите стек с `rabbitmq` и `java-supervisor` (см. [`docker-compose.yml`](../../docker-compose.yml)) и убедитесь, что в конфиге backend заданы `clients.rabbitmq` и `clients.supervisor.base_url` (как в `config.local.yaml` для Docker-сети).

После первого успешного запуска в рантайме супервизор может выставить идентификатор в поле **`orch_id`** эксперимента — по нему UI запрашивает статус по HTTP.

## Откат

Откат миграции удаляет только данные с маркером `demo-stand-skif` (`backend/migrations/down/000008_demo_stand_seed.down.sql`).
